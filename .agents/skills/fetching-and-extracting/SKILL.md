# Fetching and extracting

## When to use
Editing `src/fetch.js`, `src/clean.js`, `src/parse-core.js`/`src/parse-pool.js`, `src/selectors.js`; touching Playwright, robots, DOM pruning, Readability, dates, pre-save cleaning, or selector validation.

## Injected knowledge
- **fetchSmart decision.** Try static `got` first; if `looksEmpty` (fewer than 5 links OR under 500 chars of body text) render with Playwright and cache `needsJs=true` per host so the choice is not re-made. `forceRender` bypasses the heuristic. `src/fetch.js:143-176@79fd5d8` (hoje `src/fetch.js@3351a58`). Why: pay the browser cost only when the raw HTML is JS-gated.
- **RENDER_PROFILES must exist** — `fetchRendered` reads `RENDER_PROFILES[profile]` (`listing`: 90s deadline, 60 scroll rounds, load-more clicks; `article`: 30s, 8 rounds, no clicks). The constant was LOST in the paralel+robot-bypass merge and every rendered fetch crashed with `RENDER_PROFILES is not defined` — JS-gated pages were never captured. `src/fetch.js:208-212@d17d599`. Verified: baseline crawl failed 4x on nodeca.github.io; after the fix the same page enriched fine.
- **Playwright + scroll INTELIGENTE.** One shared browser singleton; block `image`/`media`/`font`; `waitUntil:'domcontentloaded'` 45s; Chromium MUST launch with `--no-sandbox` (container abort = V8 fatal, exit 133, não capturável; override `CRAWLER_CHROMIUM_ARGS`). A parada do scroll é decidida por `scrollRoundDecision` (função PURA, testável sem browser): perfil `article` mantém a parada clássica por platô de altura; perfil `listing` COLHE `{href,text,datetime}` a cada rodada (delta via `window.__ncSeenLinks` — um Set no page context persiste entre evaluates; o walk do datetime sobe <=4 níveis e PARA no ancestral com >3 links, senão "empresta" a data do 1º item da lista) e para por: `piso` (>=2 itens novos datados e TODOS < `--since`; o `clickLoadMore` é PULADO — só traria mais antigo), `estagnado` (`SCROLL_STALL_CHECKS`=3 checagens seguidas sem href novo — altura NÃO é sinal confiável: feed virtualizado recicla DOM com altura ~constante) ou o teto de rodadas/deadline. A colheita volta em `fetchSmart(...).harvest` e `mergeScrollHarvest` (crawl.js) a funde no fluxo normal de piso/enfileiramento com critério CONSERVADOR: mesmo host da listagem + data parseável (nav/rodapé não têm `<time datetime>`) — é o que salva itens que o feed virtualizado descartou do HTML final. `src/fetch.js@3351a58`, `src/crawl.js@3351a58`. Verified: `test/fetch.scrollstop.test.js` (7 casos) + run #3 real (nodeweekly/aiweekly pararam por piso `--since`).
- **Abort do job atravessa o fetch.** `fetchStatic/fetchRendered/fetchSmart` aceitam `{clock, signal, sinceDate}` (deadline.js): `throwIfAborted` na ENTRADA do host-limiter impede job morto de reservar a timeline de politeness; o `signal` vai ao `got` (cancela request em voo) e é checado por rodada de scroll/click; abort NÃO conta no circuit breaker (não é falha do host); esperas de politeness/lane viram `waits` no snapshot do clock (diagnóstico do `job/timeout` no inspect). `src/fetch.js@3351a58`. Verified: run #3 (0 estouros; antes 194/194 em cascata por zumbis segurando lanes).
- **JSDOM/Readability runs in a WORKER POOL, not the main thread.** `src/parse-core.js` holds the PURE parse functions (JSDOM `extractArticle`/`readableLinks`/`probablyArticle`, plus cheerio/turndown/pure helpers) with NO db/governor imports so a worker can import it; `src/parse-worker.js` is the worker; `src/parse-pool.js` is the pool. `clean.js` is now a FACADE: re-exports the pure core AND exposes `extractArticleAsync`/`readableLinksAsync`/`probablyArticleAsync` that route the JSDOM ops to the pool (the crawl/curate callers use these). A native SIGSEGV of the JSDOM CSS parser now kills ONLY the worker: the pool respawns and the task resolves to a SAFE DEFAULT (null/empty/false) — it NEVER re-runs inline (that would risk the main process). Per-task `PARSE_TIMEOUT_MS` + fallback to inline when worker_threads is unavailable (`PARSE_IN_WORKERS=false`); `PARSE_WORKERS` sizes the pool. Two hard-won invariants: (a) a SYNC spawn failure with ZERO live workers must DRAIN the queue inline — queued tasks have no timer (only `assign` arms one), so otherwise `runParse` hangs forever (regression: `test/parse-pool.spawnfail.test.js`); (b) a BUSY worker must be `ref()`d and re-`unref()`d when idle — with all workers unref'd the event loop can empty in a parses-only window and the process exits mid-crawl. cheerio/pure ops (pruneForLLM, extractPublishedDate, linksInHtml…) stay synchronous on the main thread under `cpuParse`. `src/parse-core.js@40d0505`, `src/parse-pool.js@40d0505`, `src/clean.js@40d0505`. Verified: `test/parse-pool.test.js` (echo/unknown-op/crash-restart/timeout via a crash fixture) + live run #12 (js-yaml JS-gated page rendered, 0 SIGSEGV).
- **Pre-save cleaning is junk-SPAN based, never full-text rewrite.** `cleanArticleContent` (Flash) returns `junk_spans` (verbatim UI dirt substrings) + optional clean title/date; `applyJunkSpans` deletes exact occurrences locally and `sanityCheckCleaned` rejects implausible over-deletion (keeps original + `clean/reject` event). A first version that asked for the full cleaned text back STARVED the pipeline (18 calls, 0 commits in 8 min — huge reasoning+output); span output lands in seconds. `src/parse-core.js@40d0505` (funções puras), `src/llm.js:493-530@d17d599`, `src/crawl.js:520-560@d17d599`. Verified: full-text version hung, span version enriched fine; guard caught a real over-deletion (undici release page).
- **Article extraction order.** Readability first (>= 400 chars); else cached Pro `content_selector`; else `extractArticleViaLLM`. Curated items (needs_enrich) NEVER get roundup-split and keep the aggregator title/date. `src/clean.js:11-21@d17d599`, `src/crawl.js:430-470@d17d599`.
- **Publication date extraction.** Issue/article dates stay RAW strings (`parseDate` compares). Page-level: `extractPublishedDate(html)` (JSON-LD datePublished -> meta article:published_time -> first `<time datetime>`). LISTING item dates via `dateNearLink` chain: (1) AI-derived per-template spec (`selectors.date_selector/date_attribute/date_regex`, Flash `deriveDateSelector` reading the REAL pruned page, cached only if >= max(3, 50%) of items yield parseable dates), (2) `time[datetime]` in the link/`<li>`, (3) `[class*="date"]` in the item container (nodeweekly uses `<span class="issue-date">2026-07-02</span>` — NOT `<time>`), (4) strict text-date regex on short containers. Bad specs degrade to null, never invent dates. `src/selectors.js:62-160@d17d599`, `src/crawl.js:250-300@d17d599`. Verified: test/selectors.dates.test.js (9 cases) + live: 562 nodeweekly links paired -> only 3 issues enqueued, pagination stopped at the --since floor.
- **Reject anti-bot interstitials.** `isBlockedPage(title, text)` gates every save (Cloudflare/captcha challenge pages return 200). Keep patterns SPECIFIC. For curated items a blocked target keeps the aggregator blurb instead of being dropped. `src/parse-core.js@40d0505` (isBlockedPage), `src/crawl.js:400-425@d17d599`.
- **robots + politeness.** robots.txt cached per host, fails open; Crawl-delay honored (cap 30s); per-fetch jitter; per-host `pLimit(PER_HOST_CONCURRENCY)` + host gate. Circuit breaker: 5 falhas -> open, half-open probe, cooldown dobra. `src/fetch.js:49-137@d17d599`.
- **Selector validity thresholds** gate cache writes/self-healing: link selector >= 3 unique links; content >= 400 chars; date spec >= max(3, 50% dos itens) parseable. Below threshold, re-derive. `src/selectors.js:180-210@d17d599`, `src/crawl.js:255-285@d17d599`.
