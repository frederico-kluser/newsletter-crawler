# Fetching and extracting

## When to use
Editing `src/fetch.js`, `src/clean.js`, `src/selectors.js`; touching Playwright, robots, DOM pruning, Readability, dates, pre-save cleaning, or selector validation.

## Injected knowledge
- **fetchSmart decision.** Try static `got` first; if `looksEmpty` (fewer than 5 links OR under 500 chars of body text) render with Playwright and cache `needsJs=true` per host so the choice is not re-made. `forceRender` bypasses the heuristic. `src/fetch.js:143-176@79fd5d8` (hoje `src/fetch.js:308-327@d17d599`). Why: pay the browser cost only when the raw HTML is JS-gated.
- **RENDER_PROFILES must exist** — `fetchRendered` reads `RENDER_PROFILES[profile]` (`listing`: 90s deadline, 60 scroll rounds, load-more clicks; `article`: 30s, 8 rounds, no clicks). The constant was LOST in the paralel+robot-bypass merge and every rendered fetch crashed with `RENDER_PROFILES is not defined` — JS-gated pages were never captured. `src/fetch.js:208-212@d17d599`. Verified: baseline crawl failed 4x on nodeca.github.io; after the fix the same page enriched fine.
- **Playwright.** One shared browser singleton; block `image`/`media`/`font`; `waitUntil:'domcontentloaded'` 45s; `autoScroll` até platô; `clickLoadMore` PT/EN regex. Chromium MUST launch with `--no-sandbox` (container abort = V8 fatal, exit 133, não capturável). Override via `CRAWLER_CHROMIUM_ARGS`. `src/fetch.js:74-139@79fd5d8`, `src/fetch.js:199-215@d17d599`.
- **Aggressive mode is the DEFAULT** (`AGGRESSIVE_DEFAULT`, env `CRAWLER_AGGRESSIVE=false` or `--no-aggressive` per run for polite mode; `--aggressive` forces even with env off): skips robots.txt (`ensureAllowed`) and uses a real Chrome UA + full document headers/client-hints in static AND rendered paths. Deliberately does NOT relax breaker/delays and does NOT bypass `isBlockedPage`. The TUI "não" answer must emit `no-aggressive: true` (absent now means ON). `src/config.js:106@d17d599`, `src/commands.js:170-183@d17d599`, `src/ui/screens.js:143-156@d17d599`. Verified: run log shows the default-on line; TUI test suite green.
- **pruneForLLM / cpuParse / capHtml live in `clean.js`** (moved from crawl.js so curate.js shares them without an import cycle): prune removes script/style/nav/etc and keeps `href`/`class`/`id`; cpuParse serializes SYNC JSDOM/Readability parses through the governor cpu lane; capHtml truncates >2MB HTML fail-open. `src/clean.js:20-45@d17d599`.
- **Pre-save cleaning is junk-SPAN based, never full-text rewrite.** `cleanArticleContent` (Flash) returns `junk_spans` (verbatim UI dirt substrings) + optional clean title/date; `applyJunkSpans` deletes exact occurrences locally and `sanityCheckCleaned` rejects implausible over-deletion (keeps original + `clean/reject` event). A first version that asked for the full cleaned text back STARVED the pipeline (18 calls, 0 commits in 8 min — huge reasoning+output); span output lands in seconds. `src/clean.js:38-60@d17d599`, `src/llm.js:493-530@d17d599`, `src/crawl.js:520-560@d17d599`. Verified: live run — full-text version hung, span version enriched 55 items; guard caught a real over-deletion (undici release page).
- **Article extraction order.** Readability first (>= 400 chars); else cached Pro `content_selector`; else `extractArticleViaLLM`. Curated items (needs_enrich) NEVER get roundup-split and keep the aggregator title/date. `src/clean.js:11-21@d17d599`, `src/crawl.js:430-470@d17d599`.
- **Publication date extraction.** Issue/article dates stay RAW strings (`parseDate` compares). Page-level: `extractPublishedDate(html)` (JSON-LD datePublished -> meta article:published_time -> first `<time datetime>`). LISTING item dates via `dateNearLink` chain: (1) AI-derived per-template spec (`selectors.date_selector/date_attribute/date_regex`, Flash `deriveDateSelector` reading the REAL pruned page, cached only if >= max(3, 50%) of items yield parseable dates), (2) `time[datetime]` in the link/`<li>`, (3) `[class*="date"]` in the item container (nodeweekly uses `<span class="issue-date">2026-07-02</span>` — NOT `<time>`), (4) strict text-date regex on short containers. Bad specs degrade to null, never invent dates. `src/selectors.js:62-160@d17d599`, `src/crawl.js:250-300@d17d599`. Verified: test/selectors.dates.test.js (9 cases) + live: 562 nodeweekly links paired -> only 3 issues enqueued, pagination stopped at the --since floor.
- **Reject anti-bot interstitials.** `isBlockedPage(title, text)` gates every save (Cloudflare/captcha challenge pages return 200). Keep patterns SPECIFIC. For curated items a blocked target keeps the aggregator blurb instead of being dropped. `src/clean.js:148-170@d17d599`, `src/crawl.js:400-425@d17d599`.
- **robots + politeness.** robots.txt cached per host, fails open; Crawl-delay honored (cap 30s); per-fetch jitter; per-host `pLimit(PER_HOST_CONCURRENCY)` + host gate. Circuit breaker: 5 falhas -> open, half-open probe, cooldown dobra. `src/fetch.js:49-137@d17d599`.
- **Selector validity thresholds** gate cache writes/self-healing: link selector >= 3 unique links; content >= 400 chars; date spec >= max(3, 50% dos itens) parseable. Below threshold, re-derive. `src/selectors.js:180-210@d17d599`, `src/crawl.js:255-285@d17d599`.
- **Known flake:** a rare native SIGSEGV can kill the process under parallel JSDOM load (seen once; "Could not parse CSS stylesheet" printed just before). The crawl is resumable: rerun flips in_progress->pending and curated rows keep blurbs; items accumulate idempotently. `@d17d599`.
