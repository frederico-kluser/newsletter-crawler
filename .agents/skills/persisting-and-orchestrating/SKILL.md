# Persisting and orchestrating

## When to use
Editing `src/db.js` (schema/statements), `src/crawl.js`/`src/curate.js` (frontier/curation/pagination), or `src/index.js` (loop); touching dedup or the queue.

## Injected knowledge
- **Schema.** Tables `sources`, `pages`, `articles`, `selectors`, `frontier`, `runs`, `llm_usage`, `classifications`, `article_tags`, `classification_uncovered`, **`events`** (run_id, source_id, url, stage, status, detail JSON — per-item pipeline trace read by `ncrawl inspect`; writes are fail-open via `src/events.js:7-22@d17d599` logEvent); `UNIQUE(url)`; UNIQUE `content_hash`; WAL + foreign_keys + busy_timeout. `src/db.js@d17d599`.
- **Articles quality columns (ensureColumn):** `kind` (news|tool|release), `issue_url`/`section`/`blurb` (provenance + the AGGREGATOR's own description), `content_source` ('aggregator'|'target'), `needs_enrich`, `cleaned`, `verify_status`/`verify_notes` (ok|suspect|junk + notas). Selectors gain `date_selector/date_attribute/date_regex` (AI date spec per template). All migrations idempotent via `ensureColumn`; `wipeAll` includes `events`. `src/db.js:137-186@d17d599`.
- **Data root is `NC_HOME`** (default `~/.newsletter-crawler`): SQLite DB, user `.env` (wins precedence), user `sources.json`, EXPORT_DIR. Versioned app config stays in `ROOT/config`. `src/config.js:15-88@d17d599`.
- **All SQL via one `stmts` object.** `claimNext` atomic UPDATE...RETURNING orders `retries ASC, id ASC`; `enqueue` INSERT OR IGNORE; ONE `insertArticle` serves both flows (needs_enrich distinguishes); `enrichArticle` fills target body WITHOUT touching kind/blurb/section; `requeueUrl` re-activates a done/failed job for a still-needs_enrich row. `src/db.js:68-132@79fd5d8`, `src/db.js:262-296,419-423@d17d599`.
- **Register-at-curation, enrich-at-fetch (index sources).** `processRoundup` -> `curateRoundup`: the issue page becomes Readability markdown, chunked (`CURATE_CHUNK_CHARS`) to PARALLEL Flash agents; each curated item {url,title,kind,section,blurb} is INSERTED immediately (content = title+blurb, `needs_enrich=1`, published_at = issue date) and enqueued as an article job that only ENRICHES (thin/blocked/dup-content target keeps the blurb — a tool never disappears). Sponsors/jobs never saved (LLM label + deterministic SPONSOR_RE/JOB_RE backstop). A COVERAGE pass (set difference RAW-page external links − emitted urls -> `curateLeftoverLinks` + `isRealRecoveredItem` post-filter: needs real blurb >= 30 chars, non-generic title) recovers curator omissions; the diff MUST use the raw HTML, not the Readability body — Readability can DROP real ad-adjacent item blocks (observed live: 3 headline items vanished from the body and thus never became leftovers). Event `curate/coverage` logs the URL-level funnel (leftovers/recovered/filtered/secondary). Curated items are NOT censored by the --since floor (the issue is the temporal anchor); plain articles still are. `src/curate.js:104-230@d17d599`, `src/crawl.js:360-410@d17d599`. Verified: 3 nodeweekly issues -> ~20 itens/issue com kind e seção; 0 sponsors salvos; 0 docs-secundários (baseline tinha ~15).
- **Frontier state machine.** `pending -> in_progress -> done | failed`; resume: `resetInProgress()` at startup flips stale in_progress back to pending (`src/index.js:52@79fd5d8`); `bumpRetry` até MAX_RETRIES. BUDGET_EXCEEDED devolve a pending SEM consumir retry. `src/commands.js:196-216@d17d599`.
- **One `runs` row per command.** `runWithLimits`/`beginRun` opens the ledger run; crawlRun REUSES it as the delta watermark (`getBudgetState().runId`; `startDeltaRun` only as fallback). Before the fix each crawl inserted a SECOND runs row and CRASHED on robot-bypass-era DBs (`runs.command NOT NULL`). `src/commands.js:127-140@d17d599`. Verified: live crawl on the user's old-schema DB.
- **Date floor (`--since`)**: pairs listing links with item dates (see fetching-and-extracting's dateNearLink chain incl. the AI date spec), skips below-floor and STOPS pagination; roundup re-gates by issue date; curated items exempt (issue anchor). `SINCE_MAX_INDEX_PAGES` teto. Verified: 562 nodeweekly links -> 3 issues, "piso atingido".
- **Dedup**: canonical post-redirect URL short-circuit + UNIQUE url + UNIQUE content_hash (curated content = `title — blurb`, so recurring weekly blurbs with new titles don't collide). `src/crawl.js:430-445@d17d599`, `src/curate.js:200-215@d17d599`.
- **Purge per source (`ncrawl purge <nome|substr> --yes [--selectors]`)**: deletes articles(+tags/classifications via cascade), pages, frontier, events of ONE source; source row stays -> next crawl redoes from scratch (the reproducible "apague e refaça" protocol). Ambiguous/0 matches error out listing candidates. `src/commands.js:427-462@d17d599`. Verified: 3 purge+recrawl cycles.
- **Concurrency loop**: capacity = fetch+render lanes (governor) with CONCURRENCY env as hard cap; inflight Set + Promise.race; post-crawl hooks (verify -> classify/summarize) run in parallel under the llm-only profile. `src/commands.js:230-260@d17d599`.
- **Pagination stop conditions:** empty page, repeated html_hash, no next link, `added === 0` (incremental re-crawl), below-floor item, maxPages. `findNextPage`: cached next_selector -> rel=next -> ?page=N+1 -> LLM. `src/crawl.js:330-358@d17d599`.
- **Inspect/audit:** `ncrawl inspect` (default: latest CRAWL run — `getLatestCrawlRunId`, verify/classify runs have no articles) prints run header + per-stage LLM cost + per-issue item tree (kind, verdict, chars, content source) + skip reasons; `--url <substr>` dumps the event timeline; `--verbose` includes verify notes. `src/commands.js:470-560@d17d599`.
