# Persisting and orchestrating

## When to use
Editing `src/db.js` (schema/statements), `src/crawl.js` (frontier/pagination), or `src/index.js` (loop); touching dedup or the queue.

## Injected knowledge
- **Schema.** Tables `sources`, `pages`, `articles`, `selectors`, `frontier`; `UNIQUE(url)` on each URL-bearing table; `content_hash` (indexed) for cross-URL dedup; pragmas `journal_mode=WAL` and `foreign_keys=ON`. `src/db.js:10-66@79fd5d8`.
- **Data root is `NC_HOME`, not the repo.** All USER runtime state lives under `NC_HOME` (env override; default `~/.newsletter-crawler`), `mkdir`-ed at config load: the SQLite DB (`DB_PATH` default `NC_HOME/crawler.db`; a RELATIVE `DB_PATH` now resolves against `NC_HOME`, not `ROOT`), `NC_HOME/.env` (loaded AFTER the repo `.env` so it WINS — where `ncrawl key set` writes the OpenRouter key), the user `sources.json` (seeded once from the versioned `config/sources.json`, then written by `add`), and `EXPORT_DIR`. Versioned APP config (`config/models.json`, `config/taxonomy.json`) stays in `ROOT/config`. This is what lets the `npm link` bin (`ncrawl`) run from ANY directory. `src/config.js:15-88,206-207@c82d6c4`. Verified: `npm run status` from another CWD booted `~/.newsletter-crawler/crawler.db`; `npm test` green.
- **Adding a column/index change needs a migration.** `CREATE TABLE/INDEX IF NOT EXISTS` never alters an existing DB, so new columns (`frontier.depth`, `sources.type`, `sources.max_index_pages`) go through an idempotent `ensureColumn` (PRAGMA table_info -> ALTER TABLE), and promoting `idx_articles_hash` to UNIQUE checks `pragma_index_list` then DROP+CREATE (guarded). All right after the schema exec, plus in the CREATE for fresh DBs. `src/db.js@7bf9a26`. Verified: `npm run status` migrated a pre-existing DB cleanly.
- **All SQL via one `stmts` object.** `claimNext` is an atomic `UPDATE frontier SET state='in_progress' ... RETURNING *` (hands out one job at a time, safe under the async loop); `enqueue` is `INSERT OR IGNORE` (URL dedup in the queue, now also carries `depth`); upserts use `ON CONFLICT`. `src/db.js:68-132@79fd5d8`.
- **Web search layer (`ncrawl web`) — filters WITHOUT dynamic SQL.** `webSearchArticles`/`webCountArticles` share ONE `WEB_WHERE` of NULLABLE params (`@x IS NULL OR …`; NULL = filter off), so both stay single prepared statements. Facets arrive as a JSON OBJECT `{facet:[tags]}` walked by `json_each` (key=facet, value=array): OR within a facet, AND across facets via double `NOT EXISTS`. News/tool is the boolean equality `(@kind='tool') = EXISTS(tool-tag subquery)` with the tool content-types passed as a JSON param (single-sourced from taxonomy.js). Two SQL functions registered at boot: `fold` (true case/accent-insensitive matching — apply the SAME `util.foldText` to the query) and `iso_date` — REQUIRED because `published_at` is a RAW scraped string (e.g. "June 18, 2026") for which native `date()` yields NULL; ordering and date-range filters use `coalesce(iso_date(published_at), date(extracted_at))`. `src/db.js@47bfa19`. Verified: `test/web.api.test.js` green (incl. a non-ISO published_at case) against a live 200+ article DB.
- **Frontier state machine.** `pending -> in_progress -> done | failed`. Resume: `resetInProgress()` at startup flips stale `in_progress` back to `pending`. Retry: `bumpRetry` increments and re-queues until `MAX_RETRIES`, then `finish('failed')`. `src/index.js:52,70-79@79fd5d8`.
- **Multi-level archives (index -> roundup -> article).** A source with `type:'index'` treats its discovered links as `roundup` jobs (newsletter issues), whose curated external links become `article` jobs; `frontier.depth` carries the level and `MAX_CRAWL_DEPTH` bounds recursion. An article that is really a low-prose link collection is split into N (still bounded by depth). `domainSig` is path-aware for listings (host:listing:/path-template) so `/issues` and `/issues/<slug>` don't share one cached selector; articles stay one template per host. `src/crawl.js:82-102,248-293@3113fa6`, `src/util.js:43-60@3113fa6`. Verified: aiweekly crawl index(1)->roundup(10)->article.
- **Date floor (`--since`) + reset.** `--since <date>` (parsed by `parseDate`) is an inclusive floor: `crawlArchive` pairs each index link with its `<time datetime>` (`applyLinkSelectorWithDates`), skips below-floor issues and STOPS paginating at the first one (archive is newest->oldest); `processRoundup` re-gates on the issue's own date (`extractPublishedDate`); `processArticle` drops an article whose OWN date is below the floor (null date kept — its issue is already in-range, since roundups are gated before enqueuing). `SINCE_MAX_INDEX_PAGES` is the safety ceiling. The `reset`/`clean` command (`wipeAll`: child->parent DELETE order for `foreign_keys=ON`, + VACUUM; `--yes` required) wipes all data. `src/index.js@7bf9a26`, `src/crawl.js@7bf9a26`, `src/db.js@7bf9a26`. Verified: `--since 2026-06-25` stopped pagination at the floor (3 kept, 7 cut), dropped older articles; `reset --yes` zeroed all tables.
- **Concurrency loop.** `pLimit(CONCURRENCY)` plus an `inflight` Set plus `Promise.race`; the loop stops when `inflight` is empty and `claimNext` returns nothing. `--max-articles` caps further claiming (counts only `article` jobs). `src/index.js:64-92@79fd5d8`.
- **Pagination (`crawlArchive`) stop conditions:** an empty page (selector yields under the min links), a repeated `html_hash`, no next link, `depth+1 >= maxPages` (skips the final `findNextPage`, avoiding a wasted LLM call), or a below-`--since` item. `findNextPage` order: cached `next_selector` -> `<a rel="next">` -> `?page=N+1` -> LLM (caches the discovered `next_selector`). `src/crawl.js:172-246@79fd5d8`.
- **Dedup (no link fetched/registered twice).** Identity is the POST-REDIRECT canonical URL: `processArticle` computes `canonicalUrl = normalizeUrl(fetched.url)`, short-circuits via `getArticleByUrl` BEFORE extraction, and stores under it — closing the A->B redirect-alias gap. `normalizeUrl` strips `utm_*`/`ref`/`fbclid`/`gclid`, sorts params, drops hash/trailing slash. Backstops: `UNIQUE(articles.url)` + `idx_articles_hash` now UNIQUE on `content_hash` (so `INSERT OR IGNORE` also dedups identical content under concurrency) + the `getArticleByHash` check. `src/crawl.js@7bf9a26`, `src/db.js@7bf9a26`, `src/util.js:6-18@79fd5d8`. Verified: aiweekly crawl + resume -> 0 duplicate url, 0 duplicate content_hash.
