# AGENTS.md

Single source of truth for agents working in this repo. Keep it short; scoped detail + provenance live in `.agents/skills/`.

## Setup
- First run needs the Playwright browser: `npx playwright install chromium` (or `npm run link`). Without it,
  every anti-bot site (403 → Playwright render) fails with `browserType.launch: Executable doesn't exist`.
- OpenRouter key lives in `NC_HOME/.env` (`~/.newsletter-crawler/.env`), set via `ncrawl key set <k>` — NOT in the repo.
  npm scripts use `--env-file-if-exists=.env` so they run fine without a repo `.env` (NC_HOME still wins, see config.js).

## Commands
- build: none (pure ESM, Node >= 22, no build step — the Ink TUI uses `htm` tagged templates, NOT JSX, to stay build-free)
- smoke test: `npm run status` (imports every module + boots the SQLite schema)
- unit/eval: `npm test` (node:test — parseDate, extractPublishedDate, isBlockedPage, tag-search helpers, addSourceToConfig, UI menu + search-flow render)
- guided menu (TUI): `npm run ui` (or `node src/index.js` with no args in a TTY). `CRAWLER_LANG=pt|en`, `--no-input` disables it. Fontes na tela Coletar são escolhidas por **checkbox** (MultiSelect; espaço marca, Enter confirma, todas marcadas = sem flag; subconjunto emite `--sources "A,B"`). Visual unificado por tokens semânticos (`src/ui/theme.js` + `ThemeProvider`) e widgets `Panel`/`FooterHints`/`Header` (`src/ui/widgets.js`); testes de navegação usam `test/helpers/ink.js` (por label, não DOWN×N).
- run (bounded): `npm run crawl -- --max-pages 1 --max-articles 3`. Seleção de fontes: `--sources "A,B"` (lista por vírgula, nome exato ou URL; precedência sobre `--source`/`--only`; é o que o checkbox da TUI emite). Fontes de fábrica = 6 Cooperpress (`/issues`, type index) — a 1ª coleta de fonte nova vê ~600 issues: SEMPRE com `--since`/`--max-articles`. Cada run re-visita as listagens e traz só o NOVO (`--no-refresh` desliga; a paginação para na 1ª página sem itens novos). Modo agressivo (ignora robots.txt + UA de navegador real; NÃO salva página de desafio) é o **DEFAULT** — `--no-aggressive`/`CRAWLER_AGGRESSIVE=false` volta ao educado. Toda run grava em `runs` + `articles.run_id`.
- pipeline de qualidade (default ON): issues de fontes `index` são CURADAS por IA **por seção** (1 agente Flash por News/Tools/Releases/… em paralelo, `splitIntoSections`) em itens `kind` news|tool|release com o blurb do agregador (cadastro na curadoria; o corpo do alvo vira enriquecimento — `needs_enrich`, e alvo raso/bloqueado NÃO perde o item); conteúdo extraído passa por limpeza IA pré-save (`sanityCheckCleaned` anti-truncamento) e por um guard de TEXTO PURO no armazenamento (`ensurePlainText` — o "HTML cru na UI" era content salvo com tags, nunca render); verificação (ok|suspect|junk), classificação de tags e resumos PT-BR rodam **em streaming** logo após cada enriquecimento (`streamPostSave`, + sweeps finais de rede de segurança, delta-only). Classify usa **modelo por faceta** (`classify:<faceta>` em models.json / env `LLM_MODEL_CLASSIFY_<FACETA>`): só as facetas **core** `domain`+`topic-technology` no Pro/high; as outras 7 em Flash/medium, sobre **título+início do corpo** (`CLASSIFY_MAX_CHARS`=2000) — classificação é tarefa de vocabulário fixo (small-output → Flash basta). Custo: classify já foi ~92% do gasto; este perfil corta ~4× (o ganho maior seria LOTE de artigos/1-chamada — deferido). Desliga: `CURATE_ROUNDUPS`/`CLEAN_BEFORE_SAVE`/`VERIFY_AFTER_CRAWL`/`{VERIFY,CLASSIFY,SUMMARIZE}_STREAMING=false`, `--no-verify`.
- robustez de paralelismo: o parse JSDOM/Readability roda num **pool de workers** (`src/parse-{core,worker,pool}.js`) — um SIGSEGV mata só o worker, o pool respawna e a task cai num default seguro (`PARSE_IN_WORKERS=false` força inline; `PARSE_WORKERS`/`PARSE_TIMEOUT_MS`). **Deadline por job** de artigo (`JOB_TIMEOUT_MS`, 90s; roundup/listing isentos): estourou, a ficha fica com o blurb e re-enfileira no próximo crawl. Curadoria (listing/roundup) tem **pool de reivindicação próprio** (`claimNextArticle`/`claimNextCurate`, set `curating`, `CURATE_JOBS`) p/ a fase LLM longa não travar o fetch dos artigos; a lane llm no crawl é `0.6n` (piso 3). Eventos gravados **em lote** (buffer transacional, flush no `finally` de `runWithLimits`). Custo de IA **ao vivo**: **dashboard da TUI** (fases com barras + feed curado de eventos + gasto/chamadas/por etapa; `v` abre o log cru — `src/ui/CrawlDashboard.js`, camada de marcos `src/run-events.js`) e linha `gasto parcial` no CLI (`COST_LOG_INTERVAL_MS`); a web mostra o acumulado (`apiMeta.cost`).
- auditoria: `npm run inspect` (última run: itens por issue, vereditos, motivos de skip; `--run N`, `--url <substr>`, `--verbose`) — lê a tabela `events` (trace por estágio, fail-open). `ncrawl reclean [--limit N]` re-limpa os `suspect` com passe FORTE (Pro, stage `articleReclean`) e re-verifica. `npm run purge -- <fonte> --yes [--selectors]` apaga os DADOS de uma fonte (ela continua cadastrada) p/ refazer do zero.
- verificação (ok/suspect/junk), resumos PT-BR (`title_pt`/`summary_pt`) e tags (classificação) rodam DENTRO do crawl (streaming + sweeps pós-crawl) e no `npm run finish` — NÃO há mais comando solto `verify`/`classify`/`summarize` (o `finish` cobre os três nos pendentes; desligar no crawl: `--no-verify`/`--no-classify`/`--no-summarize`).
- retomar/terminar (parou no meio?): re-rodar `npm run crawl` faz `resetInProgress` (in_progress→pending), drena a fila e roda os sweeps; ou `npm run finish -- [--budget USD] [--parallel N] [--limit N] [--no-verify|--no-classify|--no-summarize]` termina os **PENDENTES** (verify+classify+summarize) **sem novo crawl** (também no menu da TUI). Tudo delta/idempotente; `--budget` para no teto e devolve os pendentes (**retomável**). No perfil `llm-only` a lane llm = `--parallel` (`MAX_PARALLEL`) direto — mas um 429 corta a lane pela metade sozinho, então a vazão útil é logo abaixo do 1º 429.
- search: `npm run search -- <consulta> --mode B` (por tags, 5 Pro) ou `--mode A --limit N --yes` (Flash, varre o escopo). Default = só a última run QUE TROUXE ARTIGOS (âncora `MAX(articles.run_id)`; o escopo agora chega de verdade ao motor); `--all` = acervo todo. Na TUI os resultados são navegáveis: ↑/↓ seleciona, Enter abre preview (conteúdo completo rolável), `o` abre o link no browser, Esc/b volta; o confirm do modo A mostra contagem DO ESCOPO + ~US$. Durante a busca, o TUI mostra barra de progresso + % + ETA + "não analisados" ao vivo (nível-artigo via `getSearchProgress`; o modo B não move o progresso → fica indeterminado).
- web UI (buscador): `npm run web` (ou `ncrawl web [--port N] [--no-open]`) — servidor local zero-build (React UMD + htm de node_modules, API JSON sobre o SQLite) em `http://localhost:8477`. A busca DIGITADA é 100% IA (`POST /api/search`): soft (default) = 1 Flash xhigh por ~40 artigos (título+resumo; `SEARCH_BATCH_SIZE`); toggle "Busca profunda" = 1 Flash por artigo (conteúdo), escopo por fontes+período e confirmação com contagem+US$ (guards `SEARCH_MODE_A_CONFIRM`/`SEARCH_SOFT_CONFIRM` re-validados no servidor: 428 sem confirm, 409 se ocupado; teto `SEARCH_WEB_MAX_ITEMS`). A busca faz STREAMING (SSE `GET /api/search/stream`; cliente `streamSearch` via **fetch ABORTÁVEL**, NÃO EventSource — lê 428/409 e cancela): cards entram AO VIVO + loader com barra/%/X-de-Y artigos/relevantes/custo/ETA/"não analisados"; o `POST /api/search` (resposta única) segue p/ compat. Sem key → modal valida (probe) e salva em NC_HOME/.env SEM reiniciar (`setRuntimeKey`, live binding ESM). O browse sem consulta segue SQL (fonte/faceta/período/kind — kind aceita release; a busca por texto/`fold` foi REMOVIDA), dark+light. O webapp ESTÁTICO (`webapp/`) ganhou concorrência ADAPTATIVA AIMD (`webapp/src/lib/lane.js` + `adaptivePool`: corta ½ no 429, recupera +1/10s; teto via `meta.search.concurrency` — RE-EXPORTE p/ ativar no deploy) + streaming de cards. O webapp é **bilíngue PT/EN** (detecção pelo browser: pt→pt, **resto→en**; toggle manual salvo; strings em `webapp/src/strings.js` `DICTS` + `webapp/src/i18n.jsx` `useStrings` — toda string de UI nos DOIS idiomas, paridade em `webapp/test/i18n.test.js`; detecção pura em `webapp/src/lib/locale.js`, espelhada no pré-paint do `index.html`; conteúdo do acervo NÃO traduz) e abre um **tutorial** de introdução (Apple-style, `AnimatePresence mode="wait"`) na 1ª visita (`webapp/src/components/Tutorial.jsx`, `nc-tutorial-seen`; reabre pelo botão ?). Skill dedicada: `building-the-webapp` (o porte de busca segue em `searching-the-corpus`).
- histórico de buscas IA (toda busca CONCLUÍDA é salva; reabrir = resultado CONGELADO re-hidratado, ZERO LLM; re-rodar passa pela confirmação de custo usual): tabela `searches` no SQLite (CLI/TUI/web local — hits guardam só ids+vereditos, custo real via `run_id`→llm_usage) escrita por `persistSearch`/`searchWeb`; endpoints `GET|DELETE /api/searches[/:id]` no web local + dropdown de recentes no campo e painel Histórico; TUI tem a tela **Histórico de buscas** (`src/ui/HistoryView.js`, Enter abre/`r` re-roda/`d` apaga/`x`×2 limpa). O webapp ESTÁTICO persiste no NAVEGADOR (localStorage, `webapp/src/lib/history.js`, payload versionado `{v:1}` + poda só se a quota estourar) — auto-save SEM limite, dropdown+painel iguais.
- add source: `npm run add -- <url> --name "..." [--type index|listing]` (persiste em config/sources.json). export: `npm run export -- --format md [--all]` (default = última run; `--all` = acervo todo). reset: `npm run reset -- --yes` (apaga tudo).
- publicar o site: **push na main = deploy** (Vercel Git integration, Root Directory `webapp/`; os dados são os 3 JSONs commitados em `webapp/public/data`, gerados por `export --format web`). O hook versionado `.githooks/pre-push` (instalado por `npm install` via `postinstall` → `core.hooksPath`) re-exporta o snapshot do SQLite local e auto-commita se houver dado novo — nesse caso **aborta o push** (commit criado durante o push não entra nele): repita o `git push`. Guards fail-open: export falhou → push segue; snapshot MENOR que o commitado (máquina sem o banco exportaria vazio) → restaura e segue sem commit; só o `generatedAt` volátil mudou → restaura e segue.
- skills lint/eval: `node .agents/skills/scripts/skill-lint.mjs --all` / `node .agents/skills/scripts/skill-eval.mjs --all`

## Rules (only what differs from defaults and is not tooling-guaranteed)
- ESM only; never add `axios` (supply-chain incident) — use `got`.
- Log via `util` `log`/`warn`/`errorLog`, not `console.*`.
- OpenRouter reasoning effort is `xhigh`/`high`, never `max` (DeepSeek V4 → HTTP 400).
- The project `.env` must override shell-exported vars (see `src/config.js` `loadDotEnvOverride`).
- All SQL goes through the single `stmts` object in `src/db.js`.
- The Playwright context uses `ignoreHTTPSErrors: true` (some outlets have invalid TLS cert CN, e.g. kedglobal.com).
- Full, scoped knowledge with provenance: `.agents/skills/` (start at `catalog.md`).

## Skills
Every implementation task goes through `.agents/skills/project-router`. Catalog: `.agents/skills/catalog.md`.
To evolve a skill safely: stage `<skill>/SKILL.md.next`, then `node .agents/skills/scripts/validate-skill.mjs <skill>`.
